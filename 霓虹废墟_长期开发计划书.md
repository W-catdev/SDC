# 《霓虹废墟 · Neon Ruins》长期开发计划书

> **给 AI 助手的协作开发指南**  
> 基于当前 v1.0 基线代码（game.js + index.html + styles.css）

---

## 一、项目定位与核心设计哲学

### 游戏类型
俯视角2D · 射击搜刮撤离（Tarkov-like / ZERO Sievert风格）

### 技术约束
- **单文件HTML+CSS+JS**，零外部依赖，浏览器直接运行
- 所有资源程序化生成（音效用Web Audio API合成，图形用Canvas绘制）
- localStorage 跨局持久化

### 美学方向
低饱和度电影感 · 青色霓虹点缀 · 像素艺术 · **氛围优先于刺激**

### 设计铁律（任何改动不得违背）

| # | 铁律 | 说明 |
|---|------|------|
| 1 | 搜打撤不是roguelike | 敌人稀疏、慢速、有巡逻节奏，不追死人 |
| 2 | 色彩克制 | 禁用亮红(#ff0000)/亮品红(#ff00ff)，用雾粉红、暖琥珀、雾紫红 |
| 3 | 零依赖 | 不引入任何外部库或CDN资源 |
| 4 | 确定性 | 世界生成用hash2d，避免Math.random导致的画面"蠕动" |
| 5 | 单文件 | 所有代码必须在3个文件中完成（可接受CSS内联） |

## 三、开发路线图

### 阶段一：核心体验打磨（v1.1 - v1.3）

#### v1.1 手感与反馈优化
- [ ] **武器切换动画**：添加切枪时的角色姿态变化（收枪→掏枪，约200ms）
- [ ] **装填动画**：角色静止时的弹匣更换视觉（手伸向腰间）
- [ ] **受伤方向指示**：屏幕边缘显示攻击来源方向的红色警示箭头
- [ ] **击杀确认**：敌人死亡时的慢动作帧（0.1秒时停+镜头轻微推近）
- [ ] **爆头机制**：添加头部命中判定（敌人上方1/3区域），伤害×2，飘字变红

#### v1.2 AI行为深度
- [ ] **敌人听觉系统**：枪声/爆炸会惊醒巡逻敌人（范围按武器音量）
- [ ] **尸体发现**：敌人经过尸体时进入alert状态并搜索周围
- [ ] **动态难度**：根据玩家等级调整敌人数量和类型分布
- [ ] **精英小队**：3个掠夺者+1精英组成巡逻队，有简单的队形保持

#### v1.3 物理与环境交互
- [ ] **可破坏掩体**：木质掩体可被子弹击穿（每发减HP，穿透后继续飞行但伤害衰减）
- [ ] **翻越优化**：添加翻越动画（角色跃起+落地），低掩体可翻越高掩体不可
- [ ] **油桶/气罐**：地图中散布可射击引爆的桶，造成范围伤害
- [ ] **门锁系统**：部分容器需要"撬锁"（小型QTE：在正确时机按E）

### 阶段二：内容扩展（v1.4 - v1.6）

#### v1.4 新地图与生态
- [ ] **地图变体**：3种地图主题，每局随机选择
  - 城市废墟：更多掩体、霓虹招牌密集、街道布局
  - 地下设施：狭窄走廊、通风管道（可爬行）、电力室（EMP效果增强）
  - 工业区：高架平台、起重机（可攀爬）、化学池（持续伤害区域）
- [ ] **天气系统**：辐射风暴（视野降低+持续掉血）、电磁脉冲（所有电子设备失效10秒）
- [ ] **昼夜循环**：夜间模式（视野受限，敌人眼睛发光更明显，玩家有手电筒）

#### v1.5 武器与装备深度
- [ ] **武器配件系统**：枪口（消音/制退）、握把（减后坐力）、瞄具（红点/2倍/4倍）
  - 配件从容器中搜刮获得，装备后改变武器属性
- [ ] **弹药细分**：穿甲弹（对装甲兵有效）、空尖弹（对flesh伤害+30%）、亚音速（降低警戒范围）
- [ ] **投掷物扩展**：烟雾弹（遮蔽视野）、诱饵弹（吸引敌人）、C4（定时/遥控爆炸）
- [ ] **护甲耐久**：护甲有耐久度，受击降低，0时失效

#### v1.6 新敌人类型
- [ ] **狙击手**：超远程、激光瞄准线（玩家可见红线预警）、高伤害低射速
- [ ] **无人机**：飞行单位、绕过掩体、可被击落、掉落电子元件
- [ ] **重装兵**：携带盾牌（正面无敌）、需绕后或用手雷
- [ ] **Boss**：每3局出现一次，有独特机制（召唤小怪、阶段转换、弱点暴露）

### 阶段三：元游戏与长期循环（v1.7 - v2.0）

#### v1.7 商人系统
- [ ] **基地商人**：3个NPC（武器商/医疗商/黑市商人）
  - 用战利品价值兑换货币，购买装备/弹药/情报
  - 商人好感度：完成其任务解锁更好商品
- [ ] **任务板**：每日/每周随机任务（"击杀5个隐行者"、"不带武器撤离"等）
- [ ] **黑市赌博**：用稀有元件赌武器箱（随机品质）

#### v1.8 藏身处（Hideout）
- [ ] **基地升级**：用搜刮材料升级设施
  - 医疗站：开局多带1个医疗包、医疗包恢复量+10%
  - 工作台：武器修理（降低故障率）、制作简易弹药
  - 情报中心：显示本局敌人分布热力图（战局前预览）
  - 健身房：永久提升基础负重+1kg
- [ ] **设施升级树**：每个设施5级，需要特定材料组合

#### v1.9 多人异步交互
- [ ] **尸体掠夺**：地图中散布其他玩家的"幽灵尸体"（基于真实玩家数据的AI模拟）
  - 尸体携带该玩家上一局的装备，可搜刮
  - 添加"复仇标记"：被某类敌人击杀后，下局该敌人会携带你的部分装备
- [ ] **排行榜**：本地最高分、最快撤离、最高价值撤离
- [ ] **挑战模式**：每周一个特殊规则（如：只能用近战、限时60秒、无HUD等）

#### v2.0 赛季与终局
- [ ] **赛季系统**：每赛季重置部分进度，保留外观/成就
- [ ] **Prestige系统**：满级后可"转生"，获得独特外观奖励，难度永久提升
- [ ] **剧情模式**：10个关卡串联的战役，有Boss战和剧情演出
- [ ] **沙盒模式**：无限资源、自定义参数、测试武器

## 四、技术债务与重构计划

### 已知问题
1. game.js 已超过3000行，需要模块化拆分（但保持单文件约束，用IIFE模拟模块）
2. 敌人AI的update函数过长（200+行），需拆分为子函数
3. 渲染函数drawPlayer/drawEnemy有重复代码，可提取通用像素角色绘制框架
4. 手雷系统与子弹爆炸逻辑有重复（范围伤害计算）

### 重构优先级

| 优先级 | 内容 | 说明 |
|--------|------|------|
| 🔴 高 | 提取通用"范围伤害"函数 | 供手雷/榴弹/爆炸桶/燃烧瓶复用 |
| 🔴 高 | 敌人AI状态机拆分 | 拆分为独立行为树节点 |
| 🟡 中 | 武器系统数据驱动 | JSON配置+运行时实例化 |
| 🟡 中 | 粒子系统统一Emitter | 位置/速度/生命周期/颜色渐变 |
| 🟢 低 | Canvas分层渲染 | 地面层→实体层→效果层→UI层 |

## 五、AI协作规范

### 代码风格
- 使用现有命名规范：常量全大写(`C.XXX`)、游戏状态`G.XXX`、工具函数小驼峰
- 新增系统需添加 `=== 新系统名称 ===` 分隔注释块
- 颜色必须引用`C`对象或使用符合调色板的新色值
- 所有随机必须用`hash2d`（世界生成）或封装好的`rand`函数（游戏逻辑）

### 提交格式
每次迭代需提供：
1. **版本号与变更摘要**（3-5条bullet）
2. **修改的文件片段**（用 `// === 新增/修改标记 ===` 包裹）
3. **测试清单**（"验证：..."格式）
4. **已知问题/待办**

### 禁止事项 ❌
- 引入外部库（包括CDN链接）
- 使用亮红(#ff0000)或亮品红(#ff00ff)
- 添加需要服务器支持的特性（真正的多人联机、云端存档）
- 超过3个文件（可接受将CSS内联到HTML，但建议保持分离）
- 破坏现有存档格式（localStorage key变更需做迁移逻辑）

### 优先原则
> 当功能与性能冲突时，优先保证60fps  
> 当复杂度与可维护性冲突时，优先可维护性  
> 当真实感与游戏性冲突时，优先游戏性（搜打撤的核心是"紧张感"）

## 六、附录：快速参考

### 关键数据结构
```
G.player        → 玩家状态（位置/HP/武器/库存/装备）
G.enemies[]     → 敌人数组（每个含state/hp/ai状态）
G.bullets[]     → 子弹数组（owner区分玩家/敌人）
G.containers[]  → 搜刮容器
G.groundItems[] → 地上物品
G.covers[]      → 掩体矩形
STASH.items[]   → 仓库（跨局持久）
PLAYER_PROGRESS → 角色等级经验
ACHIEVEMENT_STATE → 成就状态
```

### 关键函数
```
initWorld()      → 初始化一局游戏
update()         → 主循环更新（60fps）
render()         → 主循环渲染
tryShoot()       → 射击逻辑
damageEnemy()    → 敌人受击
openContainer()  → 打开容器
endGame()        → 结算
drawPlayer()     → 玩家绘制
drawEnemy()      → 敌人绘制
```
